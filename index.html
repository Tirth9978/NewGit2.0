Features

Git-Like CLI, From Scratch

Five core commands that replicate the essential Git workflow — config, init, add, commit, and revert.

newgit config

Set global user name and email. Stored in a config file and attached to every commit as author metadata.

newgit config --global user.name "..."

newgit init

Creates the hidden .newgit/ directory with metadata files and folder structure.

newgit init

newgit add .

Scans the working directory, copies all files into a staging area, and generates a unique snapshot ID.

newgit add .

newgit commit

Generates commit ID, saves snapshot permanently, logs message + author + timestamp + diff stats.

newgit commit -m "message"

newgit revert

Restores all files from any commit's snapshot back to the working directory. Works with any file type.

newgit revert <ID>

Line-Level Diff

Automatically counts added and removed lines during each commit operation.

+142 lines -23 lines

System Programming

Core Concepts Demonstrated

A deep dive into Linux system programming — not a toy example, but a realistic tool.

fork() & Pipes

Process Creation & IPC

Child processes handle file traversal, data copying, and ID generation. Pipes coordinate communication between parent and child for passing IDs, status codes, and error messages.

fork()pipe()dup2()waitpid()

POSIX Threads

Multithreading in setup.c

Thread 1 creates the NewGit2.0 directory and base files. Thread 2 compiles the executable in parallel. Demonstrates concurrent execution.

pthread_createpthread_join

Modern Buffered I/O

Efficient File Handling

All file operations use fopen, fread, fwrite, fprintf. A custom 4096-byte buffer handles efficient file copying and line-by-line comparison.

fopenfreadfwriteBUFFER_SIZE

Filesystem APIs

Directory & File Operations

Recursive directory creation, file traversal, copying entire folder trees, and managing hidden metadata directories using Linux system calls.

How It Works

Staging & Commit Flow

A behind-the-scenes look at what happens when you run add . and commit.

newgit add .

Scan Working Directory

Traverse all files and folders in the current directory using opendir() / readdir()

Fork Child Process

A child process is spawned via fork() to handle file copying independently

Copy to Staging Area

Files are copied into .newgit/staging/ using buffered I/O with 4096-byte chunks

Generate Snapshot ID

Unique ID generated and sent back to parent via pipe()

newgit commit -m "..."

Generate Commit ID

New unique ID created for this commit snapshot

Copy Staging → Permanent

Staged files copied into .newgit/StagingInfo/<ID>/

Calculate Diff Stats

Line-by-line comparison counts added and removed lines

Append to idInfo.txt

Log entry with ID, message, author, email, date, and diff stats

Get Started

Installation & Usage

Up and running in under a minute. Requires Linux and gcc.

1

Install

install
# Clone the repository
$ git clone https://github.com/Tirth9978/NewGit2.0.git
$ cd NewGit2.0
# Give execute permission & run setup
$ chmod +x setup.sh
$ ./setup.sh
2

Use It

workflow
$ newgit config --global user.name "Tirth"
$ newgit config --global user.email "t@ex.com"
$ cd ~/my-project
$ newgit init
$ newgit add .
$ newgit commit -m "First commit"
$ newgit revert a3f8c2

What does setup.sh do?

It compiles setup.c, runs it (which spawns threads for directory creation + compilation), then compiles the main newgit binary. If PATH is updated, you can run newgit from anywhere.
